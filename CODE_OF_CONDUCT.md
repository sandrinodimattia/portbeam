# Code Of Conduct

Portbeam should be a practical, respectful project for users and contributors.

Expected behavior:

- Be direct, kind, and constructive.
- Focus technical disagreement on facts, tradeoffs, and reproducible behavior.
- Assume good intent while still asking for clarity when something is unclear.
- Respect differing experience levels and backgrounds.

Unacceptable behavior:

- Harassment, threats, or discriminatory language.
- Personal attacks instead of technical critique.
- Publishing private information without permission.
- Repeated disruption after maintainers ask for a change in behavior.

Maintainers may remove comments, close issues, or restrict participation when
behavior makes the project harder or unsafe to work on.
